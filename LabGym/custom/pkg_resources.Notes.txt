LabGym and detectron2 presently (2025-07-28) use the pkg_resources 
package in a single module, detectron2/model_zoo/model_zoo.py.
     4	import pkg_resources
        ...
   139	    cfg_file = pkg_resources.resource_filename(
   140	        "detectron2.model_zoo", os.path.join("configs", config_path)
   141	    )

Here,
    pkg_resources 
is a modified copy of the pkg_resources dir from setuptools 80.9.0,
(copied from /site-packages).

Only a resource_filename function is required by model_zoo.py.

Prepend this "custom" dir in sys.path to load this custom pkg_resources 
instead of the genuine pkg_resources by sys.path.

This can be accomplished in LabGym/__init__.py, like

    import os
    import sys
 
    assert len(__path__) == 1  # otherwise is not anticipated        
    sys.path.insert(0, os.path.join(__path__[0], 'custom'))
